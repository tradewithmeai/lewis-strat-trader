# Publication roadmap — nested object + meta tracks

The strategy (set by the lead author, 2026-06-02) is a nested escalation: the
research escalates in rigor across tiers, and — conditionally — the *process of
doing it with AI* becomes its own research object. Each tier is designed to be
**independently shippable** so the program does not hinge on the riskiest tier.

These are works **at** undergrad/master's/PhD *levels of rigor* — papers /
preprints, not submissions to an enrolled degree. They could support a future
degree; the immediate deliverables are publications.

## Object track — the research itself

| Tier | Deliverable | Core | Status / dependency |
|---|---|---|---|
| Undergrad | **Finance paper** | The event study + pre-registered Phase 2 (reaction modelling, tradeable-edge verdict). | Ship-on-completion of Phase 2. *Likely a careful null on tradeable edge — a valid undergrad result.* **Pause here for this tier.** |
| Master's | **AI-boost paper** | The AI-augmented-research methodology; WORKLOG as evidence of human-direction-vs-AI-execution. | Robust to a null finance result (see principle 2). Drafted from the worklog as it accumulates. |
| PhD | **Unified thesis** | Both: the finance contribution + the AI-boost methodology, at thesis scope/rigor. Designed *from* the undergrad submission. | Aspirational keystone. Needs breadth + sustained original contribution beyond the single vol finding. |

## Meta track — the process itself (conditional)

Triggered only **if** the PhD thesis lands. The *process* of producing the object
track with AI is written up as a side paper, and may itself escalate
undergrad → master's → "thesis on the whole process," with the AI-boost at the
core. Most novel, least conventional to place (meta-science / AI-and-society /
HCI venues). Held as **optional/aspirational, not load-bearing.**

## Guiding principles (decided)

1. **Each tier ships independently.** Lower tiers are NOT contingent on the PhD.
   Two standalone papers even if the thesis never materialises.
2. **The AI-boost track is robust to a null finance result** — and that is the
   plan's best property. A pre-registered null with no p-hacking is a *stronger*
   augmentation story than "AI found alpha". The part the author cares most about
   does not depend on the finance winning.
3. **Meta-recursion is optional.** The first AI-augmentation case study is the
   real prize; "process about process" is elegant but not load-bearing.
4. **The WORKLOG is doubly load-bearing** — evidence base for both the AI-boost
   paper and the meta track. Document now (cheap); reconstruction later is
   impossible. Validates the documentation system.

## Sequencing

1. **Now:** finish the text signal (LLM layer) → run the pre-registered Phase 2.
2. **Then:** draft the **undergrad finance paper** (the object-track tier-1) and
   ship/preprint it. Pause object track here.
3. **In parallel / next:** draft the **master's AI-boost paper** from the
   accumulated worklog.
4. **Then:** assess PhD feasibility; if viable, design the unified thesis from
   the undergrad submission.
5. **Optional, later:** the meta-process track.
